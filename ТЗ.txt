Есть список финансовых операций двух типов: ДОХОД и РАСХОД.

Логично, что, каждому пользователю присваивается свой список операций. Поэтому необходима РЕГИСТРАЦИЯ/АВТОРИЗАЦИЯ пользователя.
Отношение пользователя и операций - МНОГИЕ к ОДНОМУ. То есть у пользователя может быть много операций, но у каждой операции только один пользователь.

Таблица пользователи
- ID,
- Логин,
- Пароль,
- Имя

Таблица операций
- ID,
- ID пользователя,
- ID типа операции,
- ID категории,
- Сумма,
- Дата,
- Комментарий

Таблица типов операций
- ID
- Слаг

Отношение операций и типов операций МНОГИЕ к ОДНОМУ. У каждого типа может быть много записей, а у каждой записи - только один тип.

Таблица категорий
- ID
- ID типа операции,
- Наименование,
- Слаг

У каждого типа операций свой пул категорий.

Добавление доходов и расходов производится через форму. Здесь нужно учесть, если нажата кнопка Добавить доход, то отображать категории дохода, тоже и про расход. 

Фильтрация
Фильтровать список можно по полям: Дата (отдельная и диапазон), Тип операции, Категория.
Тип операции - единый выбор. Категория - множественный.
Изначально в фильтре все категории. Но через иерархию по типу операций.
Если выбран тип операции, то отображается категории выбранного типа операции. 
На начальном этапе сбросить можно только все типы сразу.

Группировка
Операции можно сгруппировать по трём параметрам: по дате, по типу операции и по категории. Группировка работает вместе с фильтрацией.

Например, выбран диапазон дат - неделя. И группировка по дате. То берутся только те даты, которые входят в данный диапазон.
Если выбран фильтр по типу операции и группировка одновременно. То группируются записи по выбранному фильтру.

Запись операции можно реадактировать. Редактирование производится через форму добавления. Все поля формы заполняются по данным операции. 
Изменяются поля формы (дровера): заголовок (Добавление->Редактирование) и кнопка (Добавить->Сохранить).
